Обработка «АРМ управления событиями БИТ.Phone»

Время нахождения оператора на линии равно разности между датой начала работы в БИТ.Phone (событие «Логин») и датой окончания (событие «Логаут»).

В некоторых ситуациях нужное событие может не сохраниться в базе, например АТС может вовремя не оповестить базу о том, что оператор подключился/отключился, или программа 1С:Предприятие может аварийно закрыться. Из-за этого в отчёте могут отображаться некорректные данные. В этом случае необходимо добавить недостающие события с помощью данной обработки.

При нажатии кнопки «Получить» - в таблицу загрузятся все события операторов Битфона за выбранный период.

По умолчанию в таблице отображаются только строки периодов с ошибками, т.е. строки периодов, у которых отсутствуют либо события начала работы оператора за день, либо событие окончания.

Для отображения всех строк, в т.ч. строк периодов без ошибок - необходимо нажать кнопку «Отображать всё».

У строки с проблемой может быть указана проблема:

  1. «Отсутствует логин в начале дня»
    Данная проблема означает, что у оператора не зафиксировано начало работы за день.
    Для исправления нужно указать время начала работы в поле «Начало работы», выделить исправляемую строку и нажать кнопку добавления логина для выбранной строки.
  2. «Отсутствует логаут в конце дня»
    Данная проблема означает, что у оператора не зафиксирован конец работы за день.
    Для исправления нужно указать время конца работы в поле «Конец работы», выделить исправляемую строку и нажать кнопку добавления логаута для выбранной строки.

Можно также указать время начала/конца работы и применить исправления для всех строк таблице кнопкой «На весь период».

Для удаления события выбранной строки необходимо нажать кнопку «Удалить выбранное событие».

Чтобы добавить произвольное событие на произвольную дату - нужно нажать «Добавить событие на произвольную дату», и в открывшемся окне заполнить поля даты и события.
